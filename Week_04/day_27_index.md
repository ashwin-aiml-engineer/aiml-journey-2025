# Day 27: Gradio Applications — Client Demos

Short practice files (each ~15 minutes) to build Gradio demos for NLP, CV, audio, and multi-model interfaces.

Files:
- `day_27_01_gradio_overview.md` — quick Gradio intro and install notes
- `day_27_02_basic_interface.py` — minimal Gradio demo (text in -> label out)
- `day_27_03_inputs_outputs.md` — overview of input/output components
- `day_27_04_nlp_demos.py` — sentiment/classification/pipeline demo with fallback
- `day_27_05_cv_demos.py` — image classification/upload demo (Pillow/OpenCV stub)
- `day_27_06_blocks_api.py` — Blocks API small layout (text + image tabs)
- `day_27_07_audio_video.md` — audio/video input/output notes and quick examples
- `day_27_08_sharing_deploy.md` — sharing, Spaces, and local deploy tips
- `day_27_09_exercises.md` — 12 focused 15-min exercises
- `day_27_10_progress.md` — optional progress tracker placeholder

How to use
1. From repo root run: `python Week_04\\day_27_02_basic_interface.py` to start a local Gradio demo (if `gradio` installed).
2. Most files include safe fallbacks and instructions if `gradio` is not installed.
