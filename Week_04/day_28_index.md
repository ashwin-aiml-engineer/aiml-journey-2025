# Day 28: Month 1 Capstone — Production ML Application

Short practice files (each ~15 minutes) that map to capstone concepts: architecture, data pipelines, model management, APIs, Docker, DB, frontend integration, monitoring, CI/CD and exercises.

Files:
- `day_28_01_architecture.md` — system design and microservices notes
- `day_28_02_data_pipeline.py` — simple ETL/data validation stub
- `day_28_03_model_registry.md` — model registry and versioning patterns
- `day_28_04_api_best_practices.md` — FastAPI patterns and security notes
- `day_28_05_docker_compose.md` — Docker Compose sketch for multi-service stack
- `day_28_06_db_integration.py` — simple PostgreSQL/Redis connection demo (stubs)
- `day_28_07_frontend_integration.md` — Gradio/Streamlit integration tips
- `day_28_08_monitoring_logging.md` — logging, metrics and alerting checklist
- `day_28_09_ci_cd.md` — CI/CD quick checklist (GitHub Actions sketch)
- `day_28_10_exercises.md` — 12 concise 15-min exercises

How to use
1. Read the short note files for quick concepts.
2. Run `python Week_04\\day_28_02_data_pipeline.py` and `python Week_04\\day_28_06_db_integration.py` for safe stubs.
3. Use `day_28_10_exercises.md` to plan capstone tasks in 15-minute blocks.
